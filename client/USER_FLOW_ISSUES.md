## 1. 新手视角 (New User Perspective)

### 1.1 首次进入体验

#### 问题 1: 缺少注册流程
**当前状态:** 只有登录入口，没有注册选项
**影响:** 新用户不清楚如何创建账户
**建议:**
- 添加明确的注册流程
- 或说明钱包连接即注册

#### 问题 2: 欢迎弹窗可能被忽略
**当前状态:**
- 登录后1秒自动弹出欢迎弹窗
- 用户可以轻易关闭
- 体验券信息只在弹窗中显示

**用户困惑:**
> "我点了关闭，现在找不到那个免费体验券了"

**建议:**
- 在 TopNav 添加体验券状态指示器
- 关闭弹窗时提示 "可在个人中心找到体验券"
- 添加体验券到期提醒

#### 问题 3: 没有新手引导教程
**当前状态:** 用户直接进入交易界面，没有任何指引
**用户困惑:**
> "这些按钮是干什么的？能量是什么？等级有什么用？"

**建议:**
- 添加步骤引导 (Step-by-step onboarding)
- 关键功能首次使用时显示提示气泡
- 提供 "帮助中心" 入口

#### 问题 4: 能量系统完全没有解释
**当前状态:**
- 显示能量数字但不解释来源
- 不说明能量用途
- 不提及能量衰减机制

**用户困惑:**
> "700能量是多还是少？怎么获得？能干什么？"

**建议:**
- 添加 "能量 101" 新手教程
- 能量数字旁添加详细说明弹窗
- 在用户首次看到能量消耗提示时解释

### 1.2 等级系统理解

#### 问题 5: 等级升级路径不清晰
**当前状态:**
- 只显示 "63% → Gold"
- 不显示具体需要多少手续费
- 不估算需要多少交易量

**用户困惑:**
> "63%是什么意思？我还要交易多少才能升级？"

**建议:**
- 显示 "还需 $185 手续费升级"
- 估算 "约等于 $264,286 交易量"
- 添加进度里程碑提示

#### 问题 6: 等级权益描述模糊
**当前状态:**
| 等级 | 显示的权益 |
|------|-----------|
| Bronze | AI客服 · 基础行情 |
| Silver | AI研究员 · 深度分析 |
| Gold | 专属研究员 · 专属研报 · 加入社群 · 路演日历 |
| Diamond | VIP研究员 · 电话咨询 · 私享频道 · 路演日历 |

**用户困惑:**
> "AI研究员和专属研究员有什么区别？研报在哪看？社群怎么加入？"

**建议:**
- 每个权益添加详细说明
- 提供 "预览" 功能让用户了解内容
- 升级后主动引导使用新权益

### 1.3 研究员服务发现

#### 问题 7: 研究员服务入口隐蔽
**当前状态:**
- 需要点击右下角 AI Cat 按钮
- 再切换到 "专属研究员" 标签
- 如果等级不够会显示升级提示

**用户困惑:**
> "我听说有研究员服务，在哪里？"

**建议:**
- 在主界面添加明显的研究员入口
- 低等级用户也能预览研究员列表
- 添加 "研究员" 导航菜单项

#### 问题 8: 体验券使用流程断裂
**当前状态:**
1. 欢迎弹窗领取体验券 → 可能被关闭
2. 打开聊天面板 → 默认是 AI 标签
3. 切换到研究员标签 → 显示等级不足
4. 点击 "免费体验" → 才开始使用体验券

**建议:**
- 欢迎弹窗的 "立即体验" 直接进入研究员咨询
- 或在聊天面板顶部显示 "您有1张免费体验券"

## 2. 普通用户视角 (Regular User Perspective)

### 2.1 日常交易流程

#### 问题 9: 能量获取反馈不明显
**当前状态:**
- 交易后能量增加，但没有明显提示
- 用户不知道具体获得了多少能量
- Boost 加成效果不可视

**用户困惑:**
> "我刚才交易了，能量有增加吗？增加了多少？"

**建议:**
- 交易成功后显示 "+XX 能量" 浮动提示
- 如果有 Boost，显示加成计算过程
- 在能量历史中记录每笔交易对应的能量

#### 问题 10: 能量衰减没有预警
**当前状态:**
- 每周衰减 10%
- 没有任何提前通知
- 用户不知道什么时候会衰减

**用户困惑:**
> "为什么我的能量突然少了？"

**建议:**
- 显示 "距离下次衰减还有 X 天"
- 衰减前一天发送提醒
- 在能量历史中显示衰减记录

### 2.2 路演和直播

#### 问题 11: 路演发现困难
**当前状态:**
- 只能通过顶部滚动条或日历发现
- 滚动条一次只显示一个事件
- 日历需要点击才能打开

**用户困惑:**
> "有什么直播吗？我怎么知道什么时候有？"

**建议:**
- 添加 "即将开始" 悬浮通知
- 在交易界面侧边栏显示近期路演
- 用户关注的研究员开播时推送通知

#### 问题 12: 报名状态不持久
**当前状态:**
- 报名信息保存在组件状态中
- 刷新页面后报名状态丢失
- 没有 "我的报名" 页面

**用户困惑:**
> "我记得报名了，怎么找不到了？"

**建议:**
- 报名信息持久化到后端或 localStorage
- 添加 "我的路演" 页面
- 显示历史参与记录

#### 问题 13: 直播提醒系统不完整
**当前状态:**
- 支持添加到 Apple/Google 日历
- 但 SoDEX 本身没有推送通知
- 开播时没有站内提醒

**用户困惑:**
> "我报名了，但是忘记了开播时间"

**建议:**
- 实现浏览器推送通知
- 开播前 5 分钟站内弹窗提醒
- 支持邮件/短信提醒设置

### 2.3 研究员咨询

#### 问题 14: 咨询成本不透明
**当前状态:**
- 基础咨询 10 能量
- 延续对话 5 能量/轮
- 但这些信息藏在深层 UI

**用户困惑:**
> "一次咨询要多少能量？我的能量够用几次？"

**建议:**
- 在开始咨询前明确显示费用
- 显示 "您的能量可进行 X 次咨询"
- 添加费用计算器

#### 问题 15: 研究员选择缺乏依据
**当前状态:**
- 匹配后显示多个研究员供选择
- 每个只显示名字、头衔、初始回答
- 没有历史评分、专长领域等信息

**用户困惑:**
> "这几个研究员谁更好？我应该选谁？"

**建议:**
- 显示研究员评分和评价数
- 显示专长领域标签
- 显示历史准确率（如有）

#### 问题 16: 订阅系统不完整
**当前状态:**
- 有 "订阅研究员" 概念
- 但没有订阅入口
- 不清楚订阅的好处

**用户困惑:**
> "怎么订阅研究员？订阅后有什么好处？"

**建议:**
- 在研究员卡片添加订阅按钮
- 说明订阅权益（如优先匹配、专属内容）
- 添加订阅管理页面

### 2.4 日历使用

#### 问题 17: 日历没有分类筛选
**当前状态:**
- 宏观事件和路演混在一起
- 不能按类型筛选（央行/经济/财报/加密）
- 不能按资产筛选（BTC/ETH/SOL相关）

**用户困惑:**
> "我只关心BTC相关的事件，怎么筛选？"

**建议:**
- 添加事件类型筛选器
- 添加资产关联筛选
- 支持自定义关注列表

#### 问题 18: 移动端日历体验差
**当前状态:**
- 日历在移动端隐藏
- 只能看事件列表
- 无法快速浏览整月事件

**建议:**
- 添加移动端友好的日历视图
- 支持左右滑动切换日期
- 添加周视图选项

## 3. 研究员视角 (Researcher Perspective)

### 3.1 研究员端功能缺失

#### 问题 19: 没有研究员后台
**当前状态:**
- 平台只有用户端
- 没有研究员管理后台
- 研究员如何接单、回复不清楚

**影响:**
- 无法实际测试研究员工作流
- 研究员体验完全空白

**建议:**
- 规划研究员后台系统
- 包括接单、回复、收益管理等功能

#### 问题 20: 直播功能只有模拟
**当前状态:**
- 直播页面是模拟的
- 没有真实的推流能力
- 研究员无法实际开播

**建议:**
- 集成第三方直播服务（如腾讯云直播）
- 或设计直播间管理界面

### 3.2 研究员展示

#### 问题 21: 缺少研究员主页
**当前状态:**
- 研究员只在匹配时出现
- 没有个人主页
- 无法查看历史内容

**建议:**
- 每个研究员有专属主页
- 展示观点、历史表现、用户评价
- 支持用户浏览和关注

#### 问题 22: 研究员激励不透明
**当前状态:**
- 不清楚研究员如何获得收益
- 不知道评分如何影响匹配
- 没有研究员等级系统

**建议:**
- 设计研究员激励机制
- 评分影响曝光和收益
- 添加研究员等级/认证

## 4. 关键问题汇总

### 4.1 流程断裂点 (Dead Ends)

| 场景 | 断裂点 | 用户感受 |
|------|--------|---------|
| 新用户领取体验券 | 关闭弹窗后找不到 | 困惑、失望 |
| 报名路演 | 刷新页面后状态丢失 | 不信任 |
| 想了解研究员 | 必须先消耗能量/体验券 | 犹豫、流失 |
| 想看历史咨询 | 没有记录功能 | 无法回顾 |
| 想管理订阅 | 没有订阅管理入口 | 功能缺失感 |

### 4.2 信息黑洞 (Information Gaps)

| 信息 | 缺失位置 | 影响 |
|------|---------|------|
| 能量用途 | 全平台 | 新用户不知道能量价值 |
| 能量衰减 | 全平台 | 用户突然发现能量减少 |
| 升级具体条件 | 等级面板 | 用户不知道还差多少 |
| 咨询费用 | 咨询入口 | 用户担心消耗 |
| 研究员质量 | 匹配页面 | 用户难以选择 |

### 4.3 体验痛点 (Pain Points)

| 痛点 | 严重程度 | 影响用户群 |
|------|---------|-----------|
| 没有新手引导 | 高 | 所有新用户 |
| 能量系统不透明 | 高 | 所有用户 |
| 报名不持久 | 中 | 活跃用户 |
| 研究员信息不足 | 中 | 付费用户 |
| 移动端体验差 | 中 | 移动用户 |
| 没有通知系统 | 中 | 所有用户 |

## 6. 改善方案

### 6.1 核心问题诊断

基于用户流程分析，产品存在三大核心问题：

| 问题 | 表现 | 根因 |
|------|------|------|
| **价值传递失败** | 用户不理解能量/等级系统 | 缺乏引导和说明 |
| **转化路径断裂** | 体验券找不到，入口隐蔽 | 信息架构混乱 |
| **留存机制缺失** | 无通知，无内容沉淀 | 缺乏唤醒机制 |

### 6.2 改善框架 (AARRR 模型)

```
获客 → 激活 → 留存 → 收入 → 传播
       ↑      ↑      ↑
      重点    重点   重点
```
